# Адаптация к смене схемы БД NetExec

PenHub читает базы NetExec, которые тот раскладывает по протоколам в рамках workspace. Разработчики NetExec иногда **переименовывают колонки или таблицы** в базах, в рамках обновлений. Чтобы это не ломало синхронизацию, ридер на стороне оператора ходит через JSON-карту: **`nxc_schema.json`**.

---

## Как работает маппинг

Каждое чтение в `nxc_updater.py` по сути:

```
SELECT * FROM <table>      →  dict(row)      →  value = _col(row, proto, table, "password")
```

- `_col(...)` резолвит **внутренний ключ** (`password`) в **реальное имя колонки** через `nxc_schema.json[proto][table][key]`. Если маппинга нет — fallback на сам ключ.
- Имя **таблицы** резолвит `_table(proto, table)` через зарезервированный ключ `__table__` (fallback — сам ключ таблицы).
- Поскольку читатель делает `SELECT *`, все колонки уже в dict — поэтому адаптация к новому имени — **просто правка JSON**, без правки кода.

```json
// nxc_schema.json (форма)
{
  "smb": {
    "users": {
      "username": "username",
      "password": "password",
      "__table__": "users"
    }
  }
}
```

---

## Что делать, если nxc сменил схему

### Колонку переименовали (например, `password` → `secret`)
Поправьте значение маппинга для нужного ключа `{proto}.{table}`:
```json
"password": "secret"
```
Код не трогать.

### Таблицу переименовали (например, `users` → `accounts`)
Добавьте ключ `__table__` в карту этой таблицы:
```json
"__table__": "accounts"
```
Имя должно быть простым SQL-идентификатором; если нет, `_table` отбрасывает его (с ошибкой) и берёт имя-ключ.

### JSON битый (не парсится)
Вся мапа падает на fallback «имена = ключи» — т.е. ведёт себя ровно как до появления маппинга. Всегда проверяйте JSON после правки.

---

## Как узнать о том, что колонки переименованы:

Первой строкой каждой ключевой таблицы `nxc_updater` сверяет **обязательные идентификационные колонки** (ip / host / username / password / имя conf-check, …) с тем, что вернул nxc. Пример расхождения:

```
[schema] smb.users: missing column(s) password — update nxc_schema.json
```

в `~/.nxc-collector.log`. Это **только лог** — синхронизация продолжается по доступным колонкам, поэтому сервис не «падает»; правьте JSON, увидев ошибку в логе. Опциональные поля (флаги уязвимостей, баннеры) намеренно **не** проверяются, чтобы не было ложных срабатываний на старых версиях nxc.

---

## Когда нужна правка кода

JSON переназначает **имена** уже существующих колонок и таблиц. Он **не**:

- **Добавляет новую сущность** — чтение новой таблицы, появившейся в nxc, — это правка кода.
- **Убирает исчезнувшую сущность** — снять флаг "Обязательное" удалённой nxc таблицы — это правка кода.
- **Трогает собственные схемы PenHub** (намеренно вне карты): `collector_vulns` (пишут *наши* nxc-модули), локальная объединённая БД (`_init_local_db`) и `meta`. Их nxc не контролирует.

**Переименование колонки/таблицы → правьте `nxc_schema.json`. Новые/удалённые данные → правьте код.**

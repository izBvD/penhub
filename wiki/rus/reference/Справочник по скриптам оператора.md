# Справочник по скриптам оператора

Справочник по четырём скриптам оператора. 
Установка — **[Установка — Клиент оператора](../install/Установка%20—%20Клиент%20оператора.md)**; пользовательские сценарии — **[Пример работы](../usage/Пример%20работы.md)**.

```
nxc_collector   →  установщик и конфигуратор
nxc_updater.py  →  движок синхронизации (cron */10 + @reboot)
nxce.py         →  офлайн-экстрактор
dicgenerat.py   →  генератор словарей для перебора хэшей в hashcat
```

---

## Конфигурационные файлы

| Файл                                      | Кем пишется                           | Содержит                                                   |
| ----------------------------------------- | ------------------------------------- | ---------------------------------------------------------- |
| `~/.nxc-collector.conf`                   | `nxc_collector`                       | `[collector]`: server, port, password, operator, workspace |
| `~/.nxc/nxc.conf`                         | `nxc_collector` (`-ws`, `--bh-setup`) | nxc-воркспейс, секция `[BloodHound]`                       |
| `~/.nxc/workspaces/<ws>/nxc-collector.db` | `nxc_updater` (pull)                  | локальная объединённая БД (читается `nxce`)                |
| `~/.nxc-collector.log`                    | cron                                  | лог синхронизации                                          |

**Auth-токен** (одинаковый в `nxc_collector` и `nxc_updater`): `sha256(password)` в hex, заголовок `X-Auth-Token`.

## 1. `nxc_collector` (bash) — установка и конфигурация

| Команда                     | Действие                                                                                                                                                                                                                                                                                                                                   |
| --------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| `--install`                 | Копирует `nxc_updater`, `nxc_collector`, `nxce`, `dicgenerat` в `/usr/local/bin` (или `~/bin`, если нет прав) + дает права `chmod +x`. Дополнительно копирует nxc-модули `collector_dc.py`, `collector_hosts.py` → `~/.nxc/modules/` (mkdir -p). Ставит cron `*/10` + `@reboot sleep 30` (только для `nxc_updater`). Повторный запуск обновляет скрипты и модули, конфиг сохраняет. |
| `--install -rm`             | Удаляет бинарники, nxc-модули из `~/.nxc/modules/` и cron-записи. Конфиг, лог, локальная БД сохраняются.                                                                                                                                                                                                                                   |
| `-ws` / `--workspace-setup` | Пишет `~/.nxc-collector.conf`: `--server`, `--port`, `--operator`, `--pass`, `--workspace`. При `--workspace` также правит `~/.nxc/nxc.conf` и создаёт nxc-воркспейс.                                                                                                                                                                      |
| `--bh-setup`                | Пишет настройки BloodHound в `~/.nxc/nxc.conf`: `--bh-ip/-login/-pass/-port/-enable`.                                                                                                                                                                                                                                                      |
| `--connection-test`         | `curl` `GET /api/workspaces` с токеном. 200 = OK · 401/403 = неверный пароль · пусто = сервер недоступен.                                                                                                                                                                                                                                  |
| `--show-options`            | Печатает конфиг, воркспейс, секцию BloodHound.                                                                                                                                                                                                                                                                                             |
| `-upd` / `--update`         | Принудительная синхронизация (запускает `nxc_updater.py`).                                                                                                                                                                                                                                                                                 |

## 2. `nxc_updater.py` — движок синхронизации

Запускается из cron. Каждый запуск: **resolve workspace → push → pull.**

**Resolve workspace.** `GET /api/workspaces`. Если воркспейса нет или он архивный — синхронизация **пропускается (exit 0)**, ничего не пушится. 

**Фаза 1 — push.** Читает все `~/.nxc/workspaces/<ws>/{proto}.db` (smb, ldap, winrm, mssql, ssh, ftp, nfs, vnc, wmi, rdp) плюс `nxc-vulns.db`, нормализует и `POST`-ит в `/api/sync`. 

**Фаза 2 — pull.** `GET` `/api/hosts | credentials | results | dpapi | custom_creds` (постранично) и перезаписывает локальную объединённую БД. **Полное обновление**:  `DELETE` + вставляет все, что получено от  сервера — поэтому всё скрытое/honeypot/архивное на сервере исчезает и локально. Сетевая ошибка откатывает весь pull, поэтому сбой никогда не вайпает локальную БД. Пишет `meta.last_pull`.

**Что читает каждый ридер:**

| Reader              | Протоколы          | Извлекает                                                                                |
| ------------------- | ------------------ | ---------------------------------------------------------------------------------------- |
| `_read_smb`         | smb                | hosts (+ флаги уязвимостей), users→creds, admin/loggedin-связи, DPAPI, шары, conf checks |
| `_read_users_proto` | ldap, winrm, mssql | hosts (+ proto-поля), users→creds, связи                                                 |
| `_read_creds_proto` | ftp, nfs, vnc, wmi | hosts, creds (vnc: pkey), связи, листинги директорий (ftp/nfs)                           |
| `_read_ssh`         | ssh                | hosts, creds, ssh-ключи, связи                                                           |
| `_read_rdp`         | rdp                | только hosts (nla, port)                                                                 |
Такая конфигурация в первую очередь связана с содержимым локальных БД.

Плюс `_read_collector_vulns` (отдельный `nxc-vulns.db`, который пишут nxc-модули `collector_dc`/`collector_hosts`) → `vuln_findings`.

**Нормализация паролей** в точности копирует серверную (LM:NT → NT; хэш пустого пароля → `<empty_password>`). Про страховочный маппинг схемы, применимый здесь, — см. **[Адаптация к смене схемы БД NetExec](Адаптация%20к%20смене%20схемы%20БД%20NetExec.md)**.

## 3. `nxce.py` — офлайн PWN3D Extractor

Читает **только** локальную объединённую БД. Воркспейс выбирается из `~/.nxc-collector.conf`. 

Возможность работать внутри протокола: `smb ldap winrm mssql ssh` или `all`. Запрос идёт по `auth_relations WHERE relation_type='admin'` (только PWN3D/админ).

| Флаг                             | Значение                                               |
| -------------------------------- | ------------------------------------------------------ |
| `-u/--user`, `-d/--domain`, `-p` | Фильтры без учёта регистра по user / domain / password |
| `-i IP/CIDR`                     | Точный IP или подсеть (CIDR фильтруется в Python)      |
| `--hash` / `--plain`             | Только hash / только plaintext-учётки                  |
| `--ip`                           | Только уникальные IP                                   |
| `--nxc`                          | Готовые команды `nxc <proto> <ip> -u … -H/-p …`        |
| `-c/--count`                     | Посчитать строки                                       |
| `-o FILE`                        | Запись в файл                                          |

**`--brute DIR` — экспорт для password-спрея.** Пишет парные файлы (логины/секреты построчно):
- `logins_P_brut.txt` / `pass_for_brute.txt` (plaintext)
- `logins_H_brut.txt` / `hashes_for_brute.txt` (NT-хэши)

Данные выбирает из: `credentials` + `custom_credentials`; пропускает guest-пользователей и пустые. Печатает готовую команду `nxc … --no-bruteforce --continue-on-success`.

## 4. `dicgenerat.py` — генератор словарей для hashcat

Ставится `nxc_collector --install` как `nxce` (копия в bin, `chmod +x`). Только stdlib. В отличие от `nxce`, берёт данные **с сервера по API** (не из локальной БД): конфиг/токен как у `nxc_updater` (`~/.nxc-collector.conf`, `sha256(password)`), тянет `GET /api/credentials?hide_guest=false`, `/api/custom_creds`, `/api/dpapi`.

По запросу скрипт автоматически выкачивает и модифицирует все лоигны в проекте (в т.ч. из LSA+SAM, dpapi, custom ...), превращая их в пароли. На практике каждый логин превращается в 8478 паролей после всех модификаций. В случае если транслит EN-RU неоднозначный (yeryomenko` (Ерёменко) — `ye`×`yo`×`e`, транслит может быть построен по разному) – максимально возможное число вариантов (в самом сложном который можно представитть логине): 135 648 паролей.


| Флаг       | Значение                                    |
| ---------- | ------------------------------------------- |
| `-ws NAME` | Воркспейс (перезаписать указанный в конфиге)                 |
| `-o DIR`   | Каталог вывода (по умолчанию: текущий)       |
| `-sb SEP`  | Срезать первый слева `SEP`(сепаратор) и всё до него     |
| `-sa SEP`  | Срезать последний `SEP`(сепаратор) справа и всё после него     |
| `-b N`     | Срезать ровно N символов слева             |
| `-a N`     | Срезать ровно N символов справа               |
| `--offline-file PATH` | Взять логины из локального файла (по одному на строку), **без обращения к API**. Конфиг/сервер не нужны; мутации те же|

Даёт два файла:
- **`<ws>_base.txt`** — уникальные plaintext-пароли (вкл. bruteforced) + логины + домены (в нижнем регистре) + логины/пароли DPAPI.
- **`<ws>_mutated.txt`** — модификации логинов: срез (`-sb/-sa/-b/-a`) → транслит EN→RU (ветвление, cap 8) → печать «на английской раскладке» → варианты регистра → хвосты из charset `0-9!@#$%&*_-.` (1/2/3 симв.) + годы `1970..now` → внешний `LC_ALL=C sort -u`.

Полученный словарь скармливается hashcat вместе с непробрученными хэшами проекта (кнопка **↓ HASHES.TXT** в HashKiller):

```bash
hashcat -a 0 -m 1000 <проект>_hashes.txt <проект>_mutated.txt
```

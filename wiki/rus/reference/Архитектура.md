- **Сервер**: Python / FastAPI / SQLite (WAL) + ванильный JS-SPA. Без бандлера, без фреймворка, без сборки. Два файла SQLite.
- **Клиент оператора**: четыре скрипта. Описаны в **[Установка — Клиент оператора](../install/Установка%20—%20Клиент%20оператора.md)** и **[Справочник по скриптам оператора](Справочник%20по%20скриптам%20оператора.md)**.

## Сервер: Shell + Modules

Серверное приложение — это **shell** (оболочка), в которой подключаются **модули**.

```
PenHub Shell
├── NXC Collector   (eager — встроен в shell, загружается сразу)
├── HashKiller      (lazy — подгружается при первом открытии)
├── Toolbox         (lazy — подгружается при первом открытии)
└── Reports         (lazy — подгружается при первом открытии)
```

- **Shell** отвечает за навигацию, авторизацию, общий layout, реестр модулей и доставку UI-фрагмента каждого модуля.
- Каждый **модуль** отвечает только за свой функционал, и модули общаются между собой только через общую БД и HTTP API — никогда напрямую.
- Добавление модуля = «реализовать `BaseModule` + зарегистрировать». Существующие модули не меняются. Чек-лист регистрации расписан в начале `penhub/app.py`.

| Модуль | Пакет | Загрузка | Иконка |
|---|---|---|---|
| NXC Collector | `modules/nxc_collector/` | Eager | 📡 зелёная |
| HashKiller | `modules/hashkiller/` | Lazy | 🗡 синяя |
| Toolbox | `modules/toolbox/` | Lazy | ⚙ оранжевая |
| Reports | `modules/reports/` | Lazy | 📄 фиолетовая |

UI lazy-модулей доставляется через `GET /api/shell/module/{id}/ui` (под авторизацией).

## Базы данных

| Файл                                      | Назначение                                                                      | Правило                                       |
| ----------------------------------------- | ------------------------------------------------------------------------------- | --------------------------------------------- |
| `collector.db`                            | Рабочая БД сервера: проекты, хосты, учётки, auth-связи, DPAPI, шары, уязвимости | Схема — только через миграции                 |
| `hashkiller.db`                           | Глобальная база NTLM-хэшей (`hk_pairs`)                                         | Схема — только через миграции                 |
| `~/.nxc/workspaces/<ws>/nxc-collector.db` | Локальная объединённая БД оператора                                             | Схему определяет `nxc_updater._init_local_db` |

- Везде **SQLite в режиме WAL** — движок не менять. Ожидайте сопутствующие `*.db-wal` / `*.db-shm`.
- Миграции **обратно-совместимые** — `ALTER TABLE` / `CREATE TABLE IF NOT EXISTS` в `collector/db.py`.
- `collector/db.py` и `collector/hashkiller_db.py` — стабильные внутренние API, используются повсюду.

## Конвейер синхронизации (серверная сторона)

Когда оператор пушит, `POST /api/sync` запускает упорядоченный конвейер (форма такая):

```
SyncPayload (JSON)
  → нормализация паролей
  → UPSERT hosts / INSERT-OR-IGNORE credentials / UPSERT auth_relations
  → DPAPI / shares / ssh_keys / conf_checks / directory_listings / vuln_findings
  → эмиссия уведомлений «новый PWN3D-хост»
  → авто-скрытие honeypot (новые учётки на honeypot-хостах)
  → обогащение доменными админами (флаг для watchlist-учёток) + уведомления «новый доменный админ»
  → фоном: подстановка взломанного plaintext из глобальной базы хэшей
```

Слияние **идемпотентно** (`INSERT OR IGNORE` / `UPSERT`), а WAL сериализует параллельные пуши, поэтому одновременный синк нескольких операторов в один проект безопасен.

## Фронтенд

SPA — это обычные файлы, отдаваемые из `static/`, загружаемые в фиксированном порядке, заданном в `collector/frontend.py`:

- **CSS**: токены дизайн-системы → DS core → `shell-*.css` (base, controls, table, sidebar, misc, login, projects) → CSS модулей.
- **JS**: React UMD (используется только для анимации входа) → DS bundle → `auth-app.js` → `shell-core.js` → `shell-projects.js` → семейство `shell-nxc-*.js` → JS модулей.

Все функции NXC Collector — **глобальные** (на них ссылаются inline-обработчики `onclick` в серверно-рендеренном HTML). Это намеренно и поэтому порядок загрузки фиксирован.

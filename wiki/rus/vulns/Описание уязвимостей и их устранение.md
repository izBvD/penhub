# Описание уязвимостей и их устранение

## 1. Удалённый доступ и повышение привилегий

### 1.1 SMBv1 — устаревший протокол SMB версии 1

**Описание.** На хосте включена поддержка протокола SMB версии 1 (1992 г.). Протокол не поддерживает современные механизмы шифрования и аутентификации. Является подспорьем для эксплуатации уязвимости MS17-010 (EternalBlue), а также ряда других атак на сетевой стек Windows.

**Риск.** Критический. Удалённое выполнение произвольного кода без предварительной аутентификации (при наличии MS17-010). 

**Рекомендации.**
- Отключить SMBv1 через PowerShell: `Set-SmbServerConfiguration -EnableSMB1Protocol $false`.
- Проверить, что ни одно ПО или сетевое хранилище не требует SMBv1 (устаревшие принтеры, MFP, NAS-устройства).
- Заблокировать порты 139/TCP и 445/TCP везде где это не требуется.

---

### 1.2 Signing OFF — SMB-подпись отключена

**Описание.** На хосте не требуется подпись SMB-пакетов (SMB Signing). Позволяет злоумышленнику, перехватить NTLM-аутентификацию жертвы (MITM) и ретранслировать её на данный хост (NTLM Relay), получив доступ с правами жертвы без знания её пароля.

**Риск.** Высокий. При наличии техники принудительной аутентификации (Coerce) позволяет скомпрометировать хост или доменный контроллер без учётных данных.

**Рекомендации.**
- Включить обязательную подпись SMB на всех хостах: `Set-SmbServerConfiguration -RequireSecuritySignature $true`.
- На доменных контроллерах подпись включается политикой: *Сетевая безопасность: цифровая подпись для сервера (всегда)*.
- Для рабочих станций применить групповую политику: *Computer Configuration → Windows Settings → Security Settings → Local Policies → Security Options → Microsoft network server: Digitally sign communications (always)*.

---

### 1.3 Spooler — служба диспетчера очереди печати запущена

**Описание.** На хосте запущена служба Print Spooler (spoolsv.exe). Служба предоставляет сетевой интерфейс, который может быть использован для принудительной аутентификации хоста на произвольном сервере (PrinterBug / SpoolSample), а также для эксплуатации уязвимости PrintNightmare с целью локального или удалённого выполнения кода.

**Риск.** Высокий (на серверах и доменных контроллерах, где печать не нужна).

**Рекомендации.**
- Остановить и отключить службу Print Spooler на всех хостах, где  не требуется печать: `Stop-Service Spooler; Set-Service Spooler -StartupType Disabled`.
- На доменных контроллерах отключение обязательно.
- Если печать необходима: применить патчи MS (CVE-2021-1675, CVE-2021-34527) и ограничить доступ к интерфейсу MS-RPRN.

---

### 1.4 MS17-010 — EternalBlue

**Идентификаторы:** CVE-2017-0144, CVE-2017-0145, CVE-2017-0143, MS17-010

**Описание.** Уязвимость в реализации протокола SMBv1 в Windows. Позволяет удалённому злоумышленнику без аутентификации выполнить произвольный код с правами SYSTEM. Использовалась в атаках WannaCry и NotPetya. Уязвимости более 7 лет, патч Microsoft вышел в марте 2017 года.

**Риск.** Критический. Полная компрометация хоста без учётных данных. 

**Рекомендации.**
- Установить обновление MS17-010 (KB4012212 и аналогичные в зависимости от версии ОС).
- Отключить SMBv1 (см. п. 1.1).
- Изолировать уязвимые хосты в отдельный сегмент до применения патча.
- Заблокировать порт 445/TCP везде где это не требуется.

---

### 1.5 SMBGhost — удалённое выполнение кода через SMBv3

**Идентификаторы:** CVE-2020-0796

**Описание.** Уязвимость переполнения буфера в обработчике сжатия данных протокола SMBv3.1.1. Позволяет удалённому злоумышленнику без аутентификации выполнить произвольный код с правами SYSTEM. Затрагивает Windows 10 версий 1903 и 1909, Windows Server 2019 версий 1903 и 1909.

**Риск.** Критический. Полная компрометация хоста без учётных данных. Вектор: сеть.

**Рекомендации.**
- Установить обновление безопасности KB4551762.
- В качестве временной меры отключить сжатие SMBv3: `Set-ItemProperty -Path "HKLM:\SYSTEM\CurrentControlSet\Services\LanmanServer\Parameters" DisableCompression -Type DWORD -Value 1`.
- Заблокировать порт 445/TCP везде где это не требуется.

---

### 1.6 PrintNightmare — удалённое выполнение кода через Print Spooler

**Идентификаторы:** CVE-2021-1675, CVE-2021-34527

**Описание.** Уязвимость в службе Print Spooler позволяет аутентифицированному пользователю домена удалённо загрузить произвольный DLL-файл на уязвимый хост и выполнить код с правами SYSTEM. Для эксплуатации достаточно любой доменной учётной записи, в том числе с минимальными правами.

**Риск.** Критический. Захват хоста.

**Рекомендации.**
- Установить обновления безопасности KB5004945 и последующие кумулятивные патчи.
- Отключить Print Spooler на всех хостах, где он не нужен, особенно на доменных контроллерах (см. п. 1.3).
- Если отключить невозможно: настроить политику *Point and Print Restrictions* и запретить установку принтеров не-администраторами: `HKLM\Software\Policies\Microsoft\Windows NT\Printers\PointAndPrint\RestrictDriverInstallationToAdministrators = 1`.

---

### 1.7 WebDAV — служба WebClient запущена

**Описание.** На хосте запущена служба WebClient (WebDAV). Это позволяет злоумышленнику инициировать принудительную NTLM-аутентификацию хоста через протокол HTTP (в отличие от стандартных Coerce-атак, использующих SMB). HTTP-аутентификацию можно ретранслировать на LDAP или другие сервисы без ограничений, связанных с SMB Signing, что делает атаку особенно опасной.

**Риск.** Высокий. В сочетании с любой Coerce-техникой позволяет скомпрометировать хост или выполнить RBCD-атаку (Resource-Based Constrained Delegation).

**Рекомендации.**
- Остановить и отключить службу WebClient на всех хостах, где WebDAV не используется: `Stop-Service WebClient; Set-Service WebClient -StartupType Disabled`.
- Проверить необходимость WebDAV для приложений (некоторые версии MS Office, SharePoint-клиенты).

---

### 1.8 noPac — эскалация до Domain Admin через подмену имени учётной записи

**Идентификаторы:** CVE-2021-42278, CVE-2021-42287

**Описание.** Две связанные уязвимости в Active Directory. CVE-2021-42278 позволяет изменить атрибут `sAMAccountName` компьютерной учётной записи так, чтобы он совпал с именем контроллера домена. CVE-2021-42287 — уязвимость в обработке Kerberos-запросов. Совместно позволяют любому аутентифицированному пользователю домена получить TGT с правами SYSTEM от имени контроллера домена и стать Domain Admin.

**Риск.** Критический. Полная компрометация Active Directory.

**Рекомендации.**
- Установить патчи ноября 2021 года (KB5008380, KB5008602 и аналогичные).
- Ограничить право создания компьютерных учётных записей: установить атрибут `ms-DS-MachineAccountQuota` в значение 0 на уровне домена.
- Настроить политику `StrongCertificateBindingEnforcement`.

---

### 1.9 Zerologon — захват учётной записи контроллера домена без пароля

**Идентификаторы:** CVE-2020-1472

**Описание.** Критическая уязвимость в реализации протокола Netlogon (MS-NRPC). Криптографическая ошибка в функции шифрования ComputeNetlogonCredential позволяет злоумышленнику в локальной сети без учётных данных установить сеанс Netlogon с контроллером домена и сбросить пароль его компьютерной учётной записи до пустого значения. Это обеспечивает полный контроль над доменом.

**Риск.** Критический. Полная компрометация Active Directory без учётных данных.

**Рекомендации.**
- Немедленно установить обновление KB4557222 (август 2020) и последующие кумулятивные обновления.
- Включить режим принудительного применения защиты Netlogon (Enforcement Mode): в реестре `HKLM\SYSTEM\CurrentControlSet\Services\Netlogon\Parameters\FullSecureChannelProtection = 1`.
- Проверить совместимость устаревших устройств (принтеры, NAS, VPN-концентраторы), использующих Netlogon.

---

## 2. Принудительная аутентификация (Coerce-атаки)

Уязвимости данного раздела используются для принудительного инициирования NTLM-аутентификации целевого хоста на сервере злоумышленника. Перехваченный хэш NTLMv2 может быть ретранслирован (Relay) на другой хост или офлайн-атакован перебором. В сочетании с отключённой подписью SMB (п. 1.2) или запущенным WebDAV (п. 1.7) позволяет скомпрометировать хосты или весь домен без знания учётных данных.

---

### 2.1 PetitPotam — принудительная аутентификация через LSARPC / EfsRPC

**Идентификаторы:** CVE-2021-36942

**Описание.** Уязвимость позволяет неаутентифицированному злоумышленнику через интерфейс MS-EFSR (EFS RPC) или MS-LSAD принудить контроллер домена или любой другой хост установить NTLM-сессию с сервером атакующего. Особенно критично для атак NTLM Relay → ADCS (Active Directory Certificate Services), что приводит к получению сертификата от имени DC и дальнейшей Pass-the-Certificate атаке.

**Риск.** Критический (при наличии ADCS). Высокий (без ADCS, но с отключённым Signing).

**Рекомендации.**
- Установить обновление KB5005413.
- Включить обязательную аутентификацию RPC на контроллерах домена (Enforcement Mode): `HKLM\SYSTEM\CurrentControlSet\Services\NETLOGON\Parameters\RequireSeal = 2`.
- Включить расширенную защиту аутентификации (EPA) на веб-сервисах ADCS.
- Включить HTTPS и EPA на ADCS Web Enrollment.
- Отключить интерфейс MS-EFSR через групповую политику, если шифрование EFS не используется.

---

### 2.2 PrinterBug — принудительная аутентификация через Print Spooler

**Идентификаторы:** CVE-2019-1040 (частично), SpoolSample

**Описание.** Уязвимость (или особенность реализации) в интерфейсе MS-RPRN позволяет любому аутентифицированному пользователю домена принудить произвольный хост с запущенным Print Spooler установить SMB-сессию на указанный сервер. Первая публично известная Coerce-техника (2018 г.).

**Риск.** Высокий. В сочетании с NTLM Relay позволяет скомпрометировать хосты или выполнить атаку Pass-the-Hash. Критичен, если жертва — контроллер домена.

**Рекомендации.**
- Отключить Print Spooler на всех хостах, где печать не используется, особенно на DC (см. п. 1.3).
- Заблокировать исходящие SMB-соединения с серверов и DC на рабочие станции и внешние адреса (egress filtering).
- Включить SMB Signing (см. п. 1.2) для нейтрализации Relay-атаки.

---

### 2.3 DFSCoerce — принудительная аутентификация через DFS-R

**Описание.** Техника принудительной аутентификации через интерфейс MS-DFSNM (Distributed File System: Namespace Management Protocol). Позволяет аутентифицированному пользователю домена принудить хост установить NTLM-сессию с произвольным сервером. Обходит некоторые защитные меры, направленные против PetitPotam.

**Риск.** Высокий. Аналогичен PetitPotam и PrinterBug.

**Рекомендации.**
- Ограничить доступ к интерфейсу MS-DFSNM через групповую политику или брандмауэр Windows.
- Отключить DFS Replication на системах, где репликация DFS не используется.
- Включить SMB Signing и EPA (см. п. 1.2 и п. 2.1).
- Заблокировать исходящие NTLM-аутентификации с контроллеров домена с помощью политики *Network Security: Restrict NTLM: Outgoing NTLM traffic to remote servers*.

---

### 2.4 ShadowCoerce — принудительная аутентификация через VSS

**Описание.** Техника принудительной аутентификации через интерфейс MS-FSRVP (File Server Remote VSS Protocol). Позволяет аутентифицированному пользователю домена принудить хост с запущенной службой File Server VSS Agent Service установить NTLM-сессию с произвольным сервером.

**Риск.** Высокий. Аналогичен другим Coerce-техникам.

**Рекомендации.**
- Отключить службу File Server VSS Agent Service на хостах, где она не нужна.
- Заблокировать доступ к интерфейсу MS-FSRVP через брандмауэр.
- Включить SMB Signing (см. п. 1.2).

---

## 3. Раскрытие учётных данных и локальные уязвимости

### 3.1 WDigest — кэширование паролей в открытом виде в LSASS

**Описание.** На хосте включена аутентификация WDigest (устаревший протокол для HTTP-аутентификации). При включённом WDigest Windows кэширует учётные данные пользователя в памяти процесса LSASS в открытом виде (plaintext). Злоумышленник, получивший доступ к LSASS (с правами SYSTEM или SeDebugPrivilege), может извлечь пароли всех пользователей, выполнивших вход на хост.

**Риск.** Высокий. Прямое раскрытие паролей в открытом виде при дампе памяти LSASS. Вектор: постэксплуатация.

**Рекомендации.**
- Отключить WDigest: в реестре `HKLM\SYSTEM\CurrentControlSet\Control\SecurityProviders\WDigest\UseLogonCredential = 0`.
- Применить через групповую политику: *Computer Configuration → Administrative Templates → MS Security Guide → WDigest Authentication*.
- Убедиться, что обновления KB2871997 (Windows 7/Server 2008 R2) установлены — они добавляют ключ реестра UseLogonCredential.

---

### 3.2 NTLMv1 — слабая аутентификация NTLM версии 1

**Описание.** Хост принимает аутентификацию по протоколу NTLMv1. Это устаревший протокол с известными криптографическими слабостями: хэш NTLMv1 может быть взломан на специализированном оборудовании (FPGA/GPU) за минуты или часы. Кроме того, NTLMv1 уязвим к relay-атакам и может быть понижен (downgrade) злоумышленником.

**Риск.** Высокий. Компрометация учётных данных пользователей, аутентифицировавшихся через данный хост.

**Рекомендации.**
- Установить минимальный уровень аутентификации NTLMv2: групповая политика *Network security: LAN Manager authentication level → Send NTLMv2 response only. Refuse LM & NTLM*.
- Реестр: `HKLM\SYSTEM\CurrentControlSet\Control\Lsa\LmCompatibilityLevel = 5`.
- Применить политику ко всему домену и контроллерам домена.

---

### 3.3 RunAsPPL — LSASS не защищён как Protected Process

**Описание.** Процесс LSASS (Local Security Authority Subsystem Service) не запущен в режиме Protected Process Light (PPL). Без PPL любой процесс с правами SYSTEM или с привилегией SeDebugPrivilege может выполнить дамп памяти LSASS и извлечь из неё хэши NTLM, Kerberos-билеты и, при включённом WDigest, пароли в открытом виде.

**Риск.** Высокий. Раскрытие учётных данных всех пользователей, выполнивших интерактивный вход на хост. Вектор: постэксплуатация.

**Рекомендации.**
- Включить защиту LSASS через реестр: `HKLM\SYSTEM\CurrentControlSet\Control\Lsa\RunAsPPL = 1` и перезагрузить сервер.
- Применить групповую политику: *Computer Configuration → Windows Settings → Security Settings → Local Policies → Security Options → Local Security Authority (LSA) protection*.
- Включить Credential Guard (Windows 10 Enterprise / Server 2016+) для более надёжной изоляции.
- Включить правила Windows Defender Attack Surface Reduction (ASR): блокировка создания дочерних процессов от LSASS.

---

### 3.4 UAC — контроль учётных записей не обеспечивает полную защиту

**Описание.** На хосте выявлена конфигурация UAC (User Account Control), при которой локальные администраторы автоматически получают полные привилегии без отображения запроса подтверждения (LocalAccountTokenFilterPolicy = 1 или UAC отключён полностью). Это позволяет злоумышленнику с учётными данными любого локального администратора получить полный доступ к системе удалённо (например, через PsExec, WMI, SMB).

**Риск.** Высокий. Упрощает горизонтальное перемещение по сети при наличии единых локальных учётных данных.

**Рекомендации.**
- Не отключать UAC. Установить максимальный уровень: *Уведомлять всегда*.
- Убедиться, что `HKLM\SOFTWARE\Microsoft\Windows\CurrentVersion\Policies\System\LocalAccountTokenFilterPolicy = 0` или ключ отсутствует.
- Отказаться от единых локальных паролей администратора на всех рабочих станциях — применить Microsoft LAPS (Local Administrator Password Solution) или аналог.
- Ограничить удалённый доступ локальных учётных записей через GPO: *Deny access to this computer from the network* для группы локальных администраторов (кроме доменных).

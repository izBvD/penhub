# Справочник по функционалу VULNS

**⚡ VULNS** в NXC Collector отображает уязвимости хостов, которые детектируют модули `collector_dc` / `collector_hosts` NetExec, плюс некоторые флаги из SMB. Рекомендации по устранению выявленных уязвимостей — на странице **[Описание уязвимостей и их устранение](Описание%20уязвимостей%20и%20их%20устранение.md)** (могут прилагаться к отчёту пентеста).

---

## Как интерпретировать результааты

- Строки = хосты, колонки = уязвимости, ячейки = **tri-state-бейдж**:
  - **YES** — уязвимо.
  - **no** — проверено, чисто.
  - **—** — нет данных / проверку не удалось выполнить.
- **`—` не значит «безопасно».** Это означает «неизвестно». Хост с `—` следует считать непроверенным, а не защищённым.

Фильтры сгруппированы:

| Группа | Проверки |
|---|---|
| **REMOTE** | SMBv1, Signing OFF, Spooler, MS17-010, SMBGhost, PrintNightmare, WebDAV, noPac, Zerologon |
| **COERCE** | PetitPotam, PrinterBug, DFSCoerce, ShadowCoerce |
| **ADMIN-ONLY** (конфиг) | WDigest, NTLMv1, RunAsPPL, UAC |

Кнопка **AND** комбинирует выбранные фильтры по логике И.

## Откуда берутся данные

```
nxc collector_dc / collector_hosts  →  ~/.nxc/workspaces/<ws>/nxc-vulns.db (collector_vulns)
   → nxc_updater читает → POST /api/sync (vuln_findings)
   → сервер хранит tri-state, при конфликте – «vulnerable-wins»
   → представление VULNS рендерит матрицу по хостам
```

Некоторые колонки (SMBv1, signing, spooler, zerologon, petitpotam) приходят также из SMB сканов хоста, сливаясь в ту же таблицу. При слиянии сохраняется в приоритете: `уязвимо (1) > чисто (0) > неизвестно (NULL)`, при равенстве берётся самая свежая запись.

# Экспорты

В PenHub реализовано три вида выгрузок: **XLSX**-таблицы (для просмотра/отчёта), **TXT**-списки (в основном, для развития атак и продвижения) и **ZIP**-архивы (парные файлы для credential-спреев).

---
## Из NXC Collector

| Экспорт         | Содержимое                                                                                    |
| --------------- | --------------------------------------------------------------------------------------------- |
| **ALL CREDS ↓** | Все *уникальные* учетные данные в проекте. Разделено на логические блоки. XLSX.               |
| **XLSX ↓**      | *Текущее* отображение таблицы (т.е. с учётом активных фильтров GUEST, UNIQ, HK-bruted). XLSX. |

Опция «UNIQ» дедуплицирует по `domain+login+password`, предпочитая plaintext хэшу и admin — loggedin. Фильтр **HK-bruted**, когда включён, подставляет известный plaintext вместо хэша перед экспортом — так в таблице оказываются взломанные пароли, а не хэши.

## Из Toolbox (Блок 2) — готовые списки

| Экспорт              | Содержимое                                                       |
| -------------------- | ---------------------------------------------------------------- |
| **ALL UNIQ LOGINS**  | Уникальные логины, в нижнем регистре, без Guest. TXT.            |
| **ALL UNIQ PASS**    | Уникальные plaintext-пароли (после HK-brute). TXT.               |
| **ALL UNIQ HASHES**  | Уникальные непробрученные NT-хэши. TXT.                          |
| **ALL UNIQ IP**      | Уникальные IP хостов. TXT.                                       |
| **NOT PWN3D IPs**    | Хосты, на которых нет авторизации с правами администратора. TXT. |
| **DOWNLOAD ARCHIVE** | ZIP из 5 парных файлов для password-спрея в nxc (см. ниже).      |

### Архив для password-спрея в nxc

`DOWNLOAD ARCHIVE`:

```
not_pwnd_ip.txt        ← цели (хосты, ещё не захваченные)
plaintext_logins.txt   ┐ парны 1:1
plaintext_passes.txt   ┘
hashes_logins.txt      ┐ парны 1:1
hashes_passes.txt      ┘
```

Строка N в `*_logins.txt` соответствует строке N в `*_passes.txt`.

## Из HashKiller

| Экспорт            | Содержимое                                                                                                                  |
| ------------------ | --------------------------------------------------------------------------------------------------------------------------- |
| **DOWNLOAD DB**    | Вся глобальная база хэшей (бэкап / передача).                                                                               |
| **HASHES.TXT**     | Непробрученные NT-хэши текущего проекта, для `hashcat -m 1000`. Дублирует функционал кнопки **ALL UNIQ HASHES** из Toolbox. |
| **EXPORT SMART**   | Все пары `hash:plain`, добавленные SMART ENRICH функционалом (`SMART=true`).                                                |
| **EXPORT WARNING** | Все конфликтующие пары `hash:plain` (`warning=true`).                                                                       |

## Создание различных списков на машине оператора

Все выгрузки из Toolbox  можно выполнить на машине оператора без подключения к серверу, из локальной объединённой БД (при ее наличии):

```bash
nxce all --nxc            # готовые команды nxc для каждого PWN3D-хоста
nxce smb --ip             # уникальные IP SMB-PWN3D-хостов
nxce --brute ./spray      # пишет парные файлы логинов/паролей/хэшей для спрея
```

nxce позволяет делать гибкие выгрузки (по логину, паролю, ip-адресу и т.д.) изучите help.

`nxce --brute` повторяет функционал `DOWNLOAD ARCHIVE` в Toolbox: пишет `logins_P_brut.txt` / `pass_for_brute.txt` (plaintext) и `logins_H_brut.txt` / `hashes_for_brute.txt` (NT-хэши), каждая пара одинаковой длины, и печатает в консоль готовую команду `nxc … --no-bruteforce --continue-on-success`. См. **[Справочник по скриптам оператора](../reference/Справочник%20по%20скриптам%20оператора.md)**.

# Пример работы

Подробное описание работы оператора.
Используйте её, чтобы понять, *когда и какой* модуль выполняет свою работу.

---

## Общая картина


- Каждый оператор запускает свой cron `nxc_updater`. Пуши идемпотентны и сериализованы WAL, поэтому одновременные синхронизации никогда не перезапишут или испортят данные.
- На **pull** каждый оператор получает **полный объединённый воркспейс** — данные всех операторов, не только свои. Каждая строка помечена оператором, который её нашёл.
- Переключение воркспейса (`-ws`) на машине оператора переключает на другую локальную БД; проекты никогда не смешиваются.

```
 alice ─┐
 bob   ─┼──push──►  проект PenHub "organisanionX"  ──pull──►  каждый оператор получает локальную БД
 carol ─┘                  (collector.db)
```

## Фаза 1 — Старт

1. **Запустить/проверить сервер** — работает и доступен. (**[Установка — Сервер](../install/Установка%20—%20Сервер.md)**)
2. **Создать проект** на странице Projects.
3. **Операторам - скачать скрипты (при необходимости) + конфиг** из Toolbox (DOWNLOAD SCRIPTS + COPY CONFIG STRING).
4. **Если есть** — **Добавить список Domain Admin Watchlist (из AD, например)**, используя Toolbox, чтобы PenHub их помечал и уведомлял, как только учетные данные найдутся.

---

## Фаза 2 —  Работа и накопление данных

Операторы используют **NetExec**. `nxc_updater` сам пушит находки и тянет объединённую картину.

В браузере:
- Следите за счётчиками в шапке и **колокольчиками уведомлений** 🏆 / ☠ по новым захватам и всплывающим доменным админам. (**[Уведомления](Уведомления.md)**)
- Просматривайте учетные данные и хосты по протоколам; помечайте **доменных/локальных админов**; скрывайте при необходимости лишние хосты в Manage Mode. (**[Модуль — NXC Collector](../modules/Модуль%20—%20NXC%20Collector.md)**)
- Вносите учетные записи из внешних источников (полученные не из nxc) через Toolbox **Custom Import**.

Для принудительной синхронизации: `nxc_collector -upd`.

---

## Фаза 3 — "Подбор" хэшей и обогащение базы

Обычно выполняется после некоторой накопленной "массе" хэшей и паролей.

В **HashKiller**:
1. **⚡ SMART ENRICH** — собрать каждый найденный plaintext пароль в глобальную базу хэшей.
2. **🕽 KILL THEM ALL** — подставить plaintext соответствующим хэшам проекта.
3. **↓ HASHES.TXT** → скачать файл с уникальными неподобранными хэшами, для hashcat → после подбора импортировать результаты в блок IMPORT HashKiller.

Взломанный plaintext сразу появляется в NXC Collector (фильтр **HK-bruted**) и обогащает списки для спрея. (**[Модуль — HashKiller](../modules/Модуль%20—%20HashKiller.md)**)

---

## Фаза 4 — Спрей паролей

Возьмите комплект для спрея из **Toolbox → DOWNLOAD ARCHIVE** (или `nxce --brute ./spray` офлайн). В нём содержатся ещё не захваченные цели плюс парные 1:1 файлы логин/пароль и логин/хэш. Запустите спрей nxc с `--no-bruteforce --continue-on-success`, затем синхронизируйте новые результаты. Повторяйте: новые захваты сокращают `not_pwnd_ip.txt`, новые учетные записи расширяют спрей. (**[Экспорты](Экспорты.md)**)

```
просмотр ──► взлом ──► спрей ──► синхронизация ──► просмотр …  
```

## Фаза 5 — Отчёт и завершение

- Используйте экспорт XLSX по  **VULNS** и **[Описание уязвимостей и их устранение](../vulns/Описание%20уязвимостей%20и%20их%20устранение.md)** как приложение к отчету по уязвимостям.
- В модуле **[Reports](../modules/Модуль%20—%20Reports.md)**: итоговая таблица учётных данных (**ALL CREDS ↓**), матрица уязвимостей (**ALL VULNS ↓**), локальные админы (**LOCAL ADMINS ↓**) и таймлайн проекта (**DOWNLOAD TIMELINE ↓**).
- **Архивируйте** проект по завершении — это замораживает его (синхронизация больше не принимается) и автоматически запускает финальный SMART Enrich, чтобы найденные plaintext пароли сохранились в глобальной базе хэшей для будущих работ.
- Проекты позже можно переместить в **Recycle** (мягкое удаление, имя зарезервировано) или в итоге окончательно удалить.

---

## Почему в веб интерфейсе используется скрытие, а не удаление

В ходе активной работы жёстко удалять синхронизированные строки бессмысленно — следующее обогащение вставит их заново. Чтобы убрать шум (гости, honeypots, дубли), используйте **фильтры GUEST/UNIQ**, **hide** и **strike**. По-настоящему удаляются только **custom-учетные данные** и **архивные проекты**. Отсутствие "жесткого" удаления – намеренно созданная логика синхронизации, а не отсутствующая функция.

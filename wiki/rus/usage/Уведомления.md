# Уведомления

PenHub сообщает вам **почти** в реальном времени, о следующих событиях: захвачен новый хост  или появилась учетная запись доменного админа из watchlist. 

![](../../images/notifications-bells.png)

---

## Два вида уведомлений

| Колокольчик          | Срабатывает, когда                                                                                          |
| -------------------- | ----------------------------------------------------------------------------------------------------------- |
| 🏆 **PWN3D**         | Хост получает свою **первую** admin-связь (PWN3D) — т.е. машину только что захватили. Одно событие на хост. |
| ☠ **Доменный админ** | Учетная запись из **Domain Admin Watchlist** найдена (появился пароль/хэш). Одно событие на учетную запись. |

Клик по записи переносит к объекту, используя глобальный поиск (hostname для PWN3D, username для доменного админа).

---

## Как считается «непрочитанное»

- Сервер ведёт **append-only журнал** событий (`GET /api/notifications`). Он никогда не правит и не удаляет — только записывает.
- «Непрочитанное» считается **в вашем браузере** из `localStorage`. Открытое уведомление помечает его прочитанным в данном браузере.
- Поскольку вся платформа использует один ключ доступа, формулировка «на браузер» фактически распространяется «на человека» — браузер каждого оператора отслеживает своё состояние прочтения, без серверной идентичности.
- Уведомления опрашивают сервер каждые **15 секунд, независимо от переключателя LIVE**, и считают только события **текущего** (открытого) проекта.

# PenHub

**PenHub** — это self-hosted веб-платформа для **операторов пентеста / red team**. Она агрегирует данные, которые [NetExec (nxc)](https://github.com/Pennyw0rth/NetExec) пишет в свои базы. 


Работает как self-hosted сервер, который вы разворачиваете во внутренней/рабочей сети. Операторы на своих машинах запускают небольшой клиентский скрипт, который по api отправляет данные из баз данных nxc на ваш локальный сервер, при этом забирает обратно актуальную копию данных, полученных от других операторов.

## Какую задачу решает?

В процессе пентеста у каждого оператора скапливается (а затем теряется) крайне много данных в консоли: сотни пар lsa/sam/dpapi и пр. PenHub превращает этот хаос в единую для всех операторов удобную платформу:

- **Один воркспейс на проект** — данные всех операторов стекаются в один проект.
- **Сложная логика представления и фильтрации учётных данных** — автоматическая дедупликация, приоритет plaintext над хэшем, watchlist доменных админов, скрытие honeypot.
- **HashKiller** — глобальная (для всех ваших проектов) база соответствий NTLM-хэш:plaintext, которая растёт каждый раз, когда какой-либо оператор находит пароль в открытом виде.
- **Подготовка списков для распыления** — создание файлов логинов/паролей/хэшей и списков целей, в один клик.
- **Сводка по уязвимостям** — матрица уязвимостей по хостам (Zerologon, PetitPotam, MS17-010, …) для итогового отчёта.
- **Итоговая таблица найденных данных** — удобное представление всех найденных данных в единой excel таблице.

## О предназначении

PenHub — это **инструмент для легитимных, санкционированных работ**. Он собирает и упорядочивает данные, которые NetExec уже получил в ходе пентеста или red-team-оценки. PenHub ничего не эксплуатирует.
PenHub ничего не отправляет и не агрегирует на сторонних серверах. Является полностью локальным self-hosted инструментом.
